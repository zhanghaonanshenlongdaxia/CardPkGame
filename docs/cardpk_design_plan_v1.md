# CardPkCocos 策划案 v1

## 1. 项目定位

一句话定位：

基于“赛博舆论战场”世界观的 1v1 回合制卡牌对战游戏。

核心体验：

- 通过不同阵营争夺叙事权
- 用热度、证据、规则压制构建策略对抗
- 以高辨识度的阵营卡组和强风格化卡面建立产品记忆点

项目目标：

- 完成首个可玩原型
- 建立 3 个首发阵营与一套中立牌体系
- 建立统一的规则、美术、数据结构基础
- 支持后续扩充卡包与阵营

## 2. 世界观

这是一个信息被高度武器化的近未来世界：

- 热度会转化为真实冲击力
- 平台规则能改变传播路径
- 证据可被具现、叠加并触发处置
- 各方争夺的不是单纯事实，而是叙事权

世界不是现实互联网的复刻，而是戏剧化的“舆论战场宇宙”。

## 3. 核心规则

### 3.1 胜利条件

- 对方声誉值降为 0
- 对方牌库为空且无法抽牌（后续扩展）

首版主胜利条件：

- 双方初始声誉值：20
- 任一方声誉值 <= 0 时失败

### 3.2 对局结构

- 1v1 回合制
- 初始手牌：4
- 每回合行动点：3
- 单位区：3 格
- 手牌上限：7

### 3.3 核心资源

- 声誉值：等同生命值
- 热度：用于扩散、爆发、资源转化
- 证据：用于标记、压制、处置

### 3.4 卡牌类型

- 单位牌：常驻场面，具有攻击和生命
- 事件牌：一次性结算
- 策略牌：持续型规则/场域牌

## 4. 回合流程

每回合分为：

1. 开始阶段
2. 抽牌阶段
3. 主阶段
4. 对抗阶段
5. 结束阶段

玩家在主阶段可：

- 使用事件牌
- 召唤 1 个单位
- 放置 1 张策略牌
- 使用主动技能

## 7. 卡组强度与环境关系

当前建议的首版环境关系：

- 热搜流：A-，高爽感快攻，但限制超额斩杀
- 控评流：B+ ~ A-，克热度体系，但不应完全锁死对手
- 实锤流：A，健康中速中轴，适合作为环境标尺

建议克制关系：

- 热搜流压制慢启动牌组
- 控评流克制热搜流
- 实锤流克制关键单位体系，并能与控评流形成拉扯

## 8. 平衡微调建议

### 热搜流

- 热搜推送：改为检视牌库顶 2 张，选 1 张加入手牌
- 榜一话题：每回合伤害触发最多 3 次

### 控评流

- 全站控评方案：敌方每回合第一次获得热度时，少获得 1 点热度
- 版面总控台：先保留现版本继续测试

### 实锤流

- 匿名爆料：基础施加 1 层证据，若控制实锤流单位则再加 1
- 全网实锤档案：先保留现版本继续测试

## 9. 数据结构草案

```ts
export type CardCamp = 'hotspot' | 'moderation' | 'evidence' | 'neutral';
export type CardType = 'unit' | 'event' | 'strategy';
export type CardRarity = 'common' | 'rare' | 'epic' | 'legendary';

export interface EffectCondition {
  type:
    | 'has_friendly_camp_unit'
    | 'heat_at_least'
    | 'target_has_evidence_at_least'
    | 'played_event_this_turn'
    | 'controller_has_camp_unit'
    | 'enemy_gained_heat_this_turn';
  value?: number;
  camp?: CardCamp;
}

export interface EffectAction {
  type:
    | 'gain_heat'
    | 'lose_heat'
    | 'draw'
    | 'discard'
    | 'deal_damage_unit'
    | 'deal_damage_player'
    | 'heal_player'
    | 'add_evidence'
    | 'apply_keyword'
    | 'modify_attack'
    | 'destroy_unit'
    | 'scry_draw'
    | 'set_trigger_limit';
  value?: number;
  target?: 'self' | 'ally_unit' | 'enemy_unit' | 'enemy_player' | 'all_enemy_units';
  keyword?: string;
  extra?: string;
}

export interface CardEffect {
  trigger: 'on_play' | 'on_turn_start' | 'on_turn_end' | 'on_attack' | 'on_damaged' | 'on_death' | 'passive';
  conditions?: EffectCondition[];
  actions: EffectAction[];
  description: string;
}

export interface CardDefinition {
  id: string;
  name: string;
  camp: CardCamp;
  type: CardType;
  rarity: CardRarity;
  cost: number;
  attack?: number;
  health?: number;
  deckLimit: number;
  tags: string[];
  text: string;
  effects: CardEffect[];
}
```

## 10. 美术规范

### 10.1 整体风格

- 赛博舆论战场
- 半写实 + 图形化 UI 元素
- 不做现实平台直接复刻
- 不做纯表情包梗图

### 10.2 卡片类型视觉区分

单位牌：

- 角色/单位立绘为主
- 卡框厚重
- 底部有攻击/生命面板

事件牌：

- 场景、动作、信息爆发瞬间为主
- 卡框更轻、更快
- 适合速度线、弹窗碎片、扩散感

策略牌：

- 系统、装置、规则网络、后台中枢为主
- 更像控制台或场域牌
- 边框几何感更强

### 10.3 阵营色调

热搜流：

- 橙红
- 亮黄
- 霓虹粉

控评流：

- 冷蓝
- 灰白
- 深青

实锤流：

- 深灰
- 黑
- 暗红
- 档案黄

中立：

- 米白
- 浅灰
- 银灰

### 10.4 稀有度视觉

普通：

- 基础边框
- 花纹极轻

稀有：

- 轻微金属感
- 名字栏更亮
- 角标装饰增加

史诗：

- 双层边框
- 阵营花纹明显
- 局部流光

传说：

- 专属复杂边框
- 阵营 Logo 更显眼
- 炫彩 / 烫金 / 浮雕 / 光效
